# Cleanup Protocol

**version:** 1.0.0
**purpose:** Deterministic cleanup of build artifacts, caches, test outputs, and temporary files — restoring a clean, reproducible workspace state.
**when to run:** Before committing, before publishing, after a failed build, or when the workspace feels cluttered.
**safety:** Always run dry-run first (`npm run protocol:cleanup -- --dry-run`). Nothing is deleted without listing it first.
**output:** List of removed items logged to stdout.

## Prerequisites

- Read access to project root
- Write access to project root (for actual cleanup)
- `.gitignore` in place (cleanup respects it — we don't delete tracked files)

---

## Steps

### 1. Dry Run — Identify What Will Be Removed

**What:** List all targets before deleting anything.
**How:**
  ```bash
  npm run protocol:cleanup -- --dry-run
  # Or manually:
  echo "=== DRY RUN ===" && \
  echo "Build artifacts:" && ls -la dist/ 2>/dev/null || echo "  (none)" && \
  echo "Coverage:" && ls -la coverage/ 2>/dev/null || echo "  (none)" && \
  echo "Test temp dirs:" && ls -la /tmp/test-probe/ /tmp/test-cursor/ /tmp/test-antigravity/ 2>/dev/null || echo "  (none)" && \
  echo "TypeScript cache:" && ls -la tsconfig.tsbuildinfo 2>/dev/null || echo "  (none)" && \
  echo "npm pack output:" && ls -la *.tgz 2>/dev/null || echo "  (none)" && \
  echo "Log files:" && find . -name "*.log" -not -path "*/node_modules/*" -not -path "*/.git/*" && \
  echo "OS artifacts:" && find . -name ".DS_Store" -not -path "*/.git/*" && \
  echo "Audit reports:" && ls -la /tmp/npm-audit-*.json 2>/dev/null || echo "  (none)"
  ```
**Success:** Full list printed. No deletion yet. Review and confirm before proceeding.

---

### 2. Remove Build Artifacts

**What:** Delete compiled TypeScript output.
**How:**
  ```bash
  rm -rf dist/
  ```
**Why safe:** Fully regenerated by `npm run build`. Never contains hand-edited files.
**Success:** `dist/` does not exist.

---

### 3. Remove Coverage Reports

**What:** Delete test coverage output.
**How:**
  ```bash
  rm -rf coverage/
  ```
**Why safe:** Fully regenerated by `npm test`. Coverage data is in CI — not needed locally between runs.
**Success:** `coverage/` does not exist.

---

### 4. Remove TypeScript Incremental Build Cache

**What:** Delete `.tsbuildinfo` if present (incremental compilation cache).
**How:**
  ```bash
  rm -f tsconfig.tsbuildinfo
  ```
**Why safe:** Auto-regenerated by `tsc --incremental`. Stale caches can mask type errors.
**Success:** No `*.tsbuildinfo` files in project root.

---

### 5. Remove Temporary Test Directories

**What:** Delete temp directories created by manual test cases (see Verification Protocol).
**How:**
  ```bash
  rm -rf /tmp/test-probe /tmp/test-cursor /tmp/test-antigravity /tmp/test-gemini
  ```
**Why safe:** Created by `nullprobe init /tmp/test-*` during manual testing. Not project source.
**Success:** `/tmp/test-probe`, `/tmp/test-cursor`, etc. do not exist.

---

### 6. Remove npm Pack Output

**What:** Delete any `.tgz` files left over from `npm pack` dry runs or local testing.
**How:**
  ```bash
  rm -f *.tgz
  ```
**Why safe:** Pack output is not tracked in git and can be regenerated with `npm pack`.
**Success:** No `.tgz` files in project root.

---

### 7. Remove OS and Editor Artifacts

**What:** Delete macOS `.DS_Store` files and any stray editor swap files.
**How:**
  ```bash
  find . -name ".DS_Store" -not -path "*/.git/*" -delete
  find . -name "*.swp" -name "*.swo" -not -path "*/.git/*" -delete
  find . -name "*~" -not -path "*/.git/*" -not -path "*/node_modules/*" -delete
  ```
**Why safe:** Auto-generated by macOS/editors. Not project files.
**Success:** No `.DS_Store` or swap files in project tree.

---

### 8. Remove Temporary Audit Reports

**What:** Delete JSON audit reports written to `/tmp` during Security Protocol runs.
**How:**
  ```bash
  rm -f /tmp/npm-audit-*.json
  ```
**Why safe:** Temporary outputs. Re-generated by `npm audit --json` at any time.
**Success:** No `npm-audit-*.json` in `/tmp`.

---

### 9. Verify Clean State

**What:** Confirm the workspace is in a clean, reproducible state.
**How:**
  ```bash
  git status
  # Expected: only source files (src/, docs/, protocols/, wiki/, .claude/, etc.)
  # No dist/, no coverage/, no *.tgz, no .DS_Store

  git diff --stat
  # Expected: only intentional changes (or empty if no work in progress)
  ```
**Success:** `git status` shows only tracked source files and any intentional in-progress changes. No build artifacts or temp files.

---

### 10. Log Cleanup

**What:** Record what was removed and when.
**How:** Print to stdout:
  ```
  ## Cleanup — YYYY-MM-DD HH:MM

  Removed:
  - dist/              (build artifacts)
  - coverage/          (test coverage)
  - tsconfig.tsbuildinfo
  - /tmp/test-probe/   (manual test output)
  - N .DS_Store files
  - <other items>

  Workspace state: CLEAN
  ```
**Success:** Log entry printed. Optionally append to `wiki/log.md` with tag `cleanup`.

---

## npm shortcut

```bash
# Dry run (safe — lists only)
npm run protocol:cleanup:dry

# Execute cleanup
npm run protocol:cleanup
```

---

## What Cleanup Does NOT Touch

The following are **never removed** by this protocol:

- `node_modules/` — managed by npm, remove only with `npm ci` or explicit intent
- `src/` — source code
- `wiki/` — knowledge base (human + AI maintained)
- `docs/` — project documentation
- `protocols/` — this runbook directory
- `.claude/`, `.cursor/`, `.agent/` — AI environment configs
- Any file tracked by git (i.e., visible in `git ls-files`)

## Success Criteria (overall)

- [ ] Dry run reviewed before executing
- [ ] All artifact categories checked (build, coverage, temp, OS, pack)

- [ ] Cleanup log printed
